# Music Store Multi-Agent Assistant

A token-efficient, intent-driven conversational AI for querying the Chinook Music Store database, built with LangGraph, LangChain, Groq, and PostgreSQL.

---

## How It Works

The system follows a strict linear pipeline — no open-ended ReAct loops:

```
User Input
   ↓
Verify Identity  ←──────────────────┐
   ↓ (verified)                     │
Load Memory              (ask again if not verified)
   ↓
Supervisor (routes by intent)
   ↓              ↓               ↓
Music          Invoice        General
Assistant      Assistant      Assistant
   ↓              ↓
Tool Node      Tool Node   (SQL queries)
   ↓              ↓
Formatter      Formatter
   ↓
Save Memory → END
```

**Why this design?**
- Every user query hits at most 3 LLM calls (supervisor + specialist + formatter)
- SQL is always executed by safe Python tool wrappers, never generated by the LLM
- Conversation memory (music preferences) persists per customer across sessions

## Project Structure

```
├── main.py                  # Entry point — interactive CLI
├── langgraph.json           # LangGraph Studio config
├── requirements.txt         # All dependencies
├── pyproject.toml           # Project metadata
├── .env                     # Your secrets (not committed)
│
├── states/
│   └── state.py             # Shared State TypedDict flowing through the graph
│
├── llm/
│   └── config.py            # Groq LLM initialization
│
├── retrieval/
│   └── db_engine.py         # PostgreSQL connection (SQLAlchemy)
│
├── tools/
│   ├── helpers.py           # customer lookup + message history cleaner
│   ├── music_tools.py       # SQL tools: albums, tracks, genres
│   └── invoice_tools.py     # SQL tools: invoices, line items, employees
│
├── nodes/
│   ├── verification.py      # Extracts + verifies customer identity
│   ├── human.py             # Interrupts graph to ask for identity
│   ├── memory.py            # Loads/saves customer music preferences
│   ├── supervisor.py        # Routes to correct specialist assistant
│   └── assistants.py        # Music, Invoice, General assistants + formatters
│
└── graph/
    ├── edges.py             # Conditional routing functions
    └── workflow.py          # Full LangGraph StateGraph assembly
```

- Python **3.12**

- **Groq API key** — get one free at [console.groq.com](https://console.groq.com)
- **uv** (recommended) or pip

## Setup Instructions

### 1. Clone the Repository

```bash
git clone <your-repo-url>
cd <repo-folder>
```

### 2. Create and Activate a Virtual Environment

**Using uv (recommended — faster):**
```bash
pip install uv
uv venv
# Windows
.venv\Scripts\activate
# macOS / Linux
source .venv/bin/activate
```

**Using standard pip:**
```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS / Linux
source .venv/bin/activate
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

### 4. Set Up the Chinook PostgreSQL Database

This project uses the **Chinook** sample music store database.

**Option A — Docker (easiest):**

```bash
# Start a PostgreSQL container
docker run -d \
  --name chinook-db \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=yourpassword \
  -e POSTGRES_DB=chinook \
  -p 5432:5432 \
  postgres:15

# Download the Chinook SQL script
# https://github.com/lerocha/chinook-database/releases
# Get: Chinook_PostgreSql.sql

# Load the data
psql -h localhost -U postgres -d chinook -f Chinook_PostgreSql.sql
```

**Option B — Manual PostgreSQL:**

1. Install PostgreSQL from [postgresql.org](https://www.postgresql.org/download/)
2. Create a database:
   ```bash
   psql -U postgres -c "CREATE DATABASE chinook;"
   ```
3. Download `Chinook_PostgreSql.sql` from [lerocha/chinook-database](https://github.com/lerocha/chinook-database/releases)
4. Load it:
   ```bash
   psql -U postgres -d chinook -f Chinook_PostgreSql.sql
   ```

### 5. Create the `.env` File

Create a file named `.env` in the project root with the following contents:

```env
# Groq API Key — https://console.groq.com
GROQ_API_KEY=your_groq_api_key_here

# PostgreSQL connection string
DATABASE_URL=postgresql://postgres:yourpassword@localhost:5432/chinook

# LangSmith tracing (optional — for LangGraph Studio)
LANGSMITH_API_KEY=your_langsmith_api_key_here
LANGSMITH_TRACING=true
LANGSMITH_PROJECT=Multi-Agent Music Store
```

> **Note:** Never commit your `.env` file. It is already listed in `.gitignore`.

## Running the Application

```bash
python main.py
```

You will see an interactive prompt:

```
==================================================
Welcome to Music Assistance! (Type 'exit' to quit)
==================================================

User >>
```

The system will first ask you to verify your identity (customer ID, email, or phone number from the Chinook database), then answer your questions.

**Example queries to try:**
```
User >> My email is luisg@embraer.com.br
User >> What albums do you have by the Rolling Stones?
User >> How much was my most recent invoice?
User >> Show me songs in the Rock genre
User >> Who is my support representative?
```

Type `exit`, `quit`, or `q` to stop.

## Running with LangGraph Studio (Visual Debugger)

LangGraph Studio lets you visualize the graph, inspect state at each step, and test queries interactively in a browser UI.

### Install the CLI

```bash
pip install "langgraph-cli[inmem]"
```

### Start the Studio Server

```bash
langgraph dev
```

This starts a local server. Open the URL shown in your terminal (usually `http://127.0.0.1:2024`) in your browser.

The graph registered is `MultiAgentMusicSystem` (defined in `langgraph.json`).

### What you can do in Studio
- See the full node graph visually
- Send test messages and watch execution flow in real time
- Inspect the state (messages, customer_id, loaded_memory) at each step
- Review all LLM calls and tool results

## Troubleshooting

**`DATABASE_URL not set in .env`**
→ Make sure your `.env` file exists in the project root and contains `DATABASE_URL`.

**`connection refused` on PostgreSQL**
→ Confirm your PostgreSQL server is running and the port/credentials in `DATABASE_URL` are correct.

**`groq.AuthenticationError`**
→ Check that `GROQ_API_KEY` in your `.env` is valid and not expired.

**Customer not found during verification**
→ Use an email, phone, or customer ID that actually exists in the Chinook database. Example email: `luisg@embraer.com.br`.

**Windows encoding error on startup**
→ The app automatically sets stdout to UTF-8 on Windows. If you still see errors, run: `set PYTHONIOENCODING=utf-8` before starting.
